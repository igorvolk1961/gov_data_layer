# Official Data Layer for AI Agents

Слой официальных данных для AI-агентов — специализированный компонент, предоставляющий доступ к нормативно-правовым актам и официальным данным с полной трассируемостью источников.

## Цель

Слой работает как **подложка знаний** для AI-агентов, а не как самостоятельный агент. Он не ведёт диалог с пользователем, не принимает решения — он предоставляет официальные данные с цитатами, ссылками на источники и оценкой уверенности, чтобы агенты могли строить на их основе обоснованные ответы.

## Что делает слой

- **Поиск официальных данных** по запросу агента (векторный + полнотекстовый поиск)
- **Нормализация** данных из разных источников в единую каноническую модель
- **Выдача с provenance** — каждая выдача содержит цитату, ссылку на источник, дату, юридический статус
- **Оценка уверенности** — честные сигналы о свежести данных, полноте покрытия, согласованности источников
- **Фоновая актуализация** — ingest worker обновляет данные по TTL, отслеживает юридический статус (действует / утратил силу)
- **Плавная деградация** — если источник недоступен, слой честно сообщает об этом, а не падает

## Архитектура

Архитектура слоя описана с использованием модели C4 (Simon Brown) на двух уровнях детализации:

- **Context** — место слоя в экосистеме AI-агентов ([диаграмма](context.md))
- **Container** — внутренняя структура слоя ([диаграмма](container.md))

Визуализация выполнена в нотации Mermaid C4.
Ключевые решения:
- **Шов адаптера** — адаптеры источников физически отделены от ядра, добавление нового источника не затрагивает ядро
- **Разделение hot/cold path** — роутер работает только с индексом/кэшем (быстрый hot path), ingest worker фоновый (медленный cold path)
- **Две оси времени** — (1) когда данные загружены (TTL инжеста), (2) какая редакция юридически действует
- **Каноническая модель** — все источники нормализуются в единую модель, что упрощает роутинг и сборку ответа

## Выбор хранилища

**Векторный индекс**: Qdrant — специализированная векторная БД, быстрый семантический поиск, payload filtering для базовой фильтрации.
**Метаданные и иерархия**: SQLite — иерархический рубрикатор (темы, регионы, ведомства), сложные SQL-запросы, транзакционность.

**Обоснование**:
- Иерархический рубрикатор требует рекурсивных запросов и JOIN, которые Qdrant не поддерживает
- SQLite достаточно для тестового задания, можно мигрировать на Postgres для production
- Разделение ответственности: Qdrant — поиск, SQLite — структура данных
