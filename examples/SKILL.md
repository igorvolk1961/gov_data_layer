# Agent Skill: Official Data Layer (ODL)

## Когда использовать

Используй ODL, когда вопрос пользователя касается:

- **Нормативно-правовых актов** — законы, кодексы, постановления, приказы
- **Официальных государственных данных** — налоги, льготы, соцподдержка, реестры
- **Юридического статуса** — действует/отменён/изменён
- **Государственных органов и ведомств** — их полномочия, структура, контакты

**Не используй ODL** для:
- Новостей и текущих событий (используй веб-поиск)
- Неофициальных источников (форумы, соцсети, блоги)
- Персональных данных конкретных лиц

## Доступные инструменты

### 1. `search_official_sources(query, context)`

Поиск официальных документов по запросу.

**Параметры:**
- `query` (str, обязательный) — поисковый запрос
- `context` (SearchContext, опциональный) — фильтры:
  - `region` — регион (федеральная/региональная)
  - `topic` — тематическая рубрика
  - `organization` — ведомство
  - `max_results` — макс. результатов (1–50, по умолч. 10)

**Возвращает:** список `SearchResult` с:
- `id`, `title`, `snippet` (цитата с контекстом)
- `url` — прямая ссылка на документ
- `source_name` — название источника
- `ingest_date` — дата загрузки в индекс
- `legal_status` — `active` / `revoked` / `modified` / `unknown`
- `confidence` — разложенные сигналы уверенности

### 2. `get_source(source_id)`

Получить полную карточку документа.

**Параметры:**
- `source_id` (str, обязательный) — ID документа

**Возвращает:** `OfficialDocument` с полными метаданными.

### 3. `list_topics(parent_id, query)`

Просмотр иерархического рубрикатора.

**Параметры:**
- `parent_id` (str, обязательный) — ID родительской рубрики (корень: `"root"`)
- `query` (str, обязательный) — поиск среди дочерних рубрик (пустая строка = все)

**Возвращает:** список `TopicNode` с:
- `id`, `name`, `parent_id`
- `description` — описание рубрики
- `child_count` — количество подрубрик
- `document_count` — количество документов

### 4. `get_toc(document_id, parent_section_id, query)`

Навигация по оглавлению документа.

**Параметры:**
- `document_id` (str, обязательный) — ID документа
- `parent_section_id` (str, обязательный) — ID родительского раздела (корень: `"root"`)
- `query` (str, обязательный) — поиск среди разделов (пустая строка = все)

**Возвращает:** список `TocNode` с:
- `id`, `document_id`, `title`
- `parent_id`, `level` (уровень вложенности)
- `child_count` — количество подразделов

## Как читать сигналы уверенности

Каждый `SearchResult` содержит `ConfidenceSignals`:

| Сигнал | Диапазон | Что означает |
|--------|----------|-------------|
| `retrieval_relevance` | 0.0–1.0 | Насколько результат релевантен запросу (косинусная близость) |
| `extraction_confidence` | 0.0–1.0 | Надёжность извлечения данных (1.0 = без LLM) |
| `data_freshness` | ISO datetime | Когда данные загружены в индекс |
| `legal_status` | enum | Юридический статус документа |
| `source_availability` | enum | Доступность источника (`available` / `degraded` / `unavailable`) |

**Рекомендуемые пороги** (настраиваются агентом):
- `retrieval_relevance` ≥ 0.7 — хорошее совпадение
- `extraction_confidence` ≥ 0.8 — надёжное извлечение
- `data_freshness` — зависит от сценария (НПА: не старше 24ч)
- `legal_status == "active"` — только действующие акты
- `source_availability != "unavailable"` — источник отвечает

## Примеры использования

### Пример 1: Поиск действующего НПА

```
Запрос пользователя: "Какая ставка НДФЛ в 2026 году?"

Действия агента:
1. search_official_sources(query="ставка НДФЛ 2026", context={topic: "налоги"})
2. Проверить legal_status == "active" у результатов
3. Если retrieval_relevance > 0.7 — использовать как источник
4. Вернуть пользователю ответ с цитатой и ссылкой
```

### Пример 2: Навигация по рубрикатору

```
Запрос пользователя: "Найди документы по налоговым льготам"

Действия агента:
1. list_topics(parent_id="root", query="налоги")
   → получаем список налоговых рубрик
2. list_topics(parent_id="налоговые_льготы", query="")
   → получаем подрубрики
3. search_official_sources(query="налоговые льготы", context={topic: "налоговые_льготы"})
```

### Пример 3: Работа с большим документом

```
Запрос пользователя: "Что сказано в НК РФ про вычет на детей?"

Действия агента:
1. search_official_sources(query="НК РФ вычет на детей")
2. get_source(source_id="nk-rf")
3. get_toc(document_id="nk-rf", parent_section_id="root", query="вычет")
   → находим нужный раздел
4. get_toc(document_id="nk-rf", parent_section_id="глава_23", query="")
   → смотрим подразделы
```

## Обработка ошибок

| Ситуация | Действие агента |
|----------|----------------|
| `NotFoundError` | Сообщить пользователю «официальный документ не найден» |
| `SourceUnavailableError` | Использовать кэшированные данные с пометкой «источник временно недоступен» |
| `InvalidInputError` | Исправить параметры запроса |
| Пустой результат поиска | Расширить запрос или переключиться на веб-поиск |
| `legal_status == "revoked"` | Не использовать, предупредить пользователя |
| `source_availability == "unavailable"` | Отказаться от ответа или указать на возможную неактуальность |

## Важно

- ODL **не заменяет** веб-поиск, а дополняет его для официальных данных
- Всегда проверяй `legal_status` — отменённые акты не должны уходить пользователю
- Свёртку сигналов уверенности в единый ответ делаешь ты (агент), слой отдаёт только факты
- Если слой вернул `found: false` — честно сообщи пользователю, не пытайся «додумать»
