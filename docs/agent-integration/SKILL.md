# Agent Skill: Official Data Layer (ODL)

## Когда использовать

Используй ODL, когда вопрос пользователя касается:

- **Законов, постановлений, приказов** — официальные документы государственных органов
- **Официальных государственных данных** — налоги, льготы, соцподдержка, реестры
- **Юридического статуса** — действует / утратил силу
- **Государственных органов и ведомств**

**Не используй ODL** для:
- Новостей и текущих событий (используй веб-поиск)
- Неофициальных источников (форумы, соцсети, блоги)
- Персональных данных конкретных лиц

## Доступные инструменты

### 1. `search_documents(query, region, organization, max_age_days, max_results, offset)`

Поиск официальных документов по текстовому запросу. Используй для поиска законов, постановлений, приказов и других официальных актов.

**Параметры (policy — задаёшь ты, агент):**

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `query` | `str` | обязательный | Свободный текст вопроса/интент пользователя |
| `region` | `str \| None` | `None` | Регион (город, область, край). Пример: «Московская область» |
| `organization` | `list[str] \| None` | `None` | Органы для фильтрации (OR). Пример: `["Минтруд России"]` |
| `max_age_days` | `int \| None` | `None` | Макс. возраст документа в днях (только для «Что нового?») |
| `max_results` | `int` | `10` | Макс. результатов (1–50, хард-кап слоя) |
| `offset` | `int` | `0` | Смещение для пагинации |

**Возвращает:** `SearchResponse`

| Поле | Тип | Описание |
|------|-----|----------|
| `results` | `list[SearchResult]` | Результаты на текущей странице |
| `total_count` | `int` | Общее количество найденных |
| `offset` | `int` | Использованное смещение |

Каждый `SearchResult`:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `str` | ID документа (формат: `{source_id}-{publish_id}`) |
| `title` | `str` | Заголовок |
| `snippet` | `str` | Цитата с релевантным контекстом |
| `url` | `str` | Прямая ссылка на документ |
| `source_name` | `str` | Название источника |
| `jurisdiction` | `str \| None` | Юрисдикция |
| `region` | `str \| None` | Регион |
| `topic` | `list[str]` | Тематические рубрики |
| `organization` | `str \| None` | Орган, принявший документ |
| `legal_status` | `active` / `not_active` | Юридический статус |
| `confidence` | `ConfidenceSignals` | Разложенные сигналы уверенности |

### 2. `get_document_detail(document_id, query, score_threshold, max_citation_length, max_chunks)`

Получить полную карточку документа с цитатами. Используй после `search_documents`, когда выбран конкретный документ.

**Параметры:**

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `document_id` | `str` | обязательный | ID документа, из `SearchResult.id` |
| `query` | `str \| None` | `None` | Фильтрация цитат по запросу (вернуть только релевантные разделы) |
| `score_threshold` | `float \| None` | `None` | Мин. порог релевантности цитат. `None` = слой использует динамический порог |
| `max_citation_length` | `int` | `2000` | Макс. суммарная длина цитат в символах |
| `max_chunks` | `int` | `5` | Макс. количество чанков для цитат |

**Возвращает:** `DocumentDetail`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `str` | ID документа |
| `title` | `str` | Заголовок |
| `url` | `str` | Прямая ссылка |
| `source_name` | `str` | Источник |
| `legal_status` | `active` / `not_active` | Юр. статус |
| `citations` | `list[Citation]` | Цитаты с путём к разделу |

Каждая `Citation`:
- `text` — текст цитаты
- `source_id` — ID документа
- `url` — ссылка
- `section` — путь к разделу (например, `["Раздел I", "Глава 2", "Статья 10"]`)
- `span_start`, `span_end` — позиция в документе

## Как читать сигналы уверенности

Каждый `SearchResult` содержит `ConfidenceSignals`:

| Сигнал | Тип | Что означает |
|--------|-----|-------------|
| `retrieval_relevance` | `float [0,1]` | Насколько текст результата релевантен запросу (косинусная близость) |
| `topic_relevance` | `list[TopicMatch] \| None` | Топ-3 рубрики, ближайшие к запросу: `[(topic_id, score)]` |
| `last_verified_at` | `datetime \| None` | Дата последней верификации данных в индексе |

**Важно:** Слой не сворачивает сигналы в одну меру. Свёртку (answerability) делаешь ты, агент, под свой сценарий.

**Рекомендуемые пороги** (настраиваются тобой):
- `retrieval_relevance ≥ 0.5` — приемлемое совпадение
- `legal_status == "active"` — только действующие акты
- Если `legal_status == "not_active"` — не используй документ, сообщи пользователю

## Примеры использования

### Пример 1: Поиск действующего документа

```
Запрос пользователя: "Какая ставка НДФЛ в 2026 году?"

Действия агента:
1. search_documents(query="ставка НДФЛ 2026")
2. Проверить legal_status == "active" у результатов
3. Выбрать результат с retrieval_relevance > 0.5
4. get_document_detail(document_id=result.id)
5. Вернуть пользователю цитату со ссылкой на источник
```

### Пример 2: Поиск с региональной фильтрацией

```
Запрос пользователя: "Какие льготы для многодетных в Московской области?"

Действия агента:
1. search_documents(
     query="льготы многодетным",
     region="Московская область"
   )
2. Проверить region в результатах — совпадает
3. Выбрать результат с legal_status == "active"
4. get_document_detail(document_id=result.id)
```

### Пример 3: Детальная карточка документа

```
Запрос пользователя: "Расскажи подробнее про приказ 668н"

Действия агента:
1. search_documents(query="приказ 668н пособия детям")
   → находим pravo-0001202012230060
2. get_document_detail(document_id="pravo-0001202012230060")
   → получаем цитаты с разделами
3. Цитируем пользователю с указанием раздела документа
```

## Обработка ошибок

| Ситуация | Действие агента |
|----------|----------------|
| `NOT_FOUND` | Сообщить: «официальный документ по вашему запросу не найден» |
| `INVALID_INPUT` | Исправить параметры запроса |
| Пустой `results` | Расширить запрос или переключиться на веб-поиск |
| `legal_status == "not_active"` | Не использовать документ, предупредить пользователя |

## Важно

- ODL **не заменяет** веб-поиск, а дополняет его для официальных данных
- Всегда проверяй `legal_status` — отменённые акты не должны уходить пользователю
- Свёртку сигналов уверенности в единый ответ делаешь ты (агент), слой отдаёт только факты
- Если документ не найден — честно сообщи пользователю, не пытайся «додумать»
- `region` и `organization` — единственные параметры, которые ты задаёшь осознанно. `score_threshold` и `topic` — механизм слоя, оставь `None`
